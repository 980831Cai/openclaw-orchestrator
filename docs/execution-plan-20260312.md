## 执行顺序

### 第一阶段：插件迁回收口

- 校验插件目录、包声明、工作区声明、安装脚本。
- 验证插件能被安装到 OpenClaw 扩展目录。
- 验证插件入口可被加载。

### 第二阶段：定时开关与核心交互

- 定位工作流页面定时开关关闭后仍在右侧的问题。
- 确认是否存在样式类失效、状态同步错误、条件渲染残留三类问题。
- 修复后补最小回归。

### 第三阶段：严格审查

- 工作流模块：编辑器、执行态、审批、定时任务、执行日志、边缘 DAG。
- 监控与交互模块：Empire、Agent 广场、聊天实时更新、WebSocket、首页状态区、活跃工作流。
- 3D 小人：状态切换、路径移动、避障表现、消息与状态同步。

### 第四阶段：分级修复

- 高严重度：阻断功能、错误状态、错误执行、数据不一致。
- 中严重度：交互误导、状态不同步、异常 UI 行为、边缘逻辑缺陷。
- 低严重度：文案、可用性、视觉细节、非阻断表现问题。

### 第五阶段：回归与收口

- 手工验证工作流创建、保存、执行、定时、审批、活跃状态展示。
- 手工验证聊天、实时流、首页监控、Agent 广场。
- 手工验证 3D 小人状态与运动表现。
